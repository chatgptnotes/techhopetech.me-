Dev Toolkit

Day-1 reading for new Bettroi inductees. The five Claude Code tools every developer installs to cut tokens, quota usage, and rate-limit hits.

Tool Purpose Key benefit
1. Knowledge Graph (Graphify) Maps the entire codebase visually and logically. Saves massive tokens by helping Claude "ride the lines" to the right file instead of searching blindly.
2. Read-Website-Fast MCP free A "strainer" for the web that turns messy HTML into clean Markdown. Scrape leads and docs without wasting tokens on ads, banners, and cookie pop-ups. No paid API key needed.
3. NotebookLM Skill Connects Claude to Google's research tool via browser cookies. Query thousands of sources (PDFs, YouTube videos) at $0 retrieval cost.
4. Awesome Design MCP Library of 68 brand identities (Apple, Claude, Linear, etc.). Gives Claude "taste" so it generates gorgeous, modern UI instead of generic-looking code.
5. Claude Code Router (ccr) API billing only Local proxy that routes tasks to different models via OpenRouter. Routes simple tasks to cheap models (DeepSeek), heavy tasks to Opus, saving up to 80% on API costs. Skip if you're on Claude MAX or Pro (flat fee). Useful only when paying per token.

Install all five in one command

From any new dev laptop:

git clone https://github.com/chatgptnotes/claude-code-deepseek-backend.git
cd claude-code-deepseek-backend
bash install-all.sh

Each tool also has a standalone script under tools/install-*.sh if you only want some of them.

Three-brain provider switch

Optional. Three terminal commands flip Claude Code between providers per session:

claude-ds       # DeepSeek (cheap)
claude-glm      # Z.AI / GLM (alt cheap)
claude-an       # Anthropic (premium)
claude-status   # which brain is active

Same caveat as ccr: this saves money only on API billing. On Claude MAX / Pro you've already paid the flat fee.

Daily flow

  • Map first. Run /graphify on any unfamiliar repo before editing.
  • Scrape clean. Use read-website-fast instead of plain WebFetch.
  • Offload retrieval. Stash long PDFs and YouTube transcripts in NotebookLM.
  • UI taste. Tell Claude "set active design to <brand>" before generating UI.
  • Route cheap (API users only). Launch via ccr code for routine boilerplate.

Why this combination

Each tool plugs a different leak:

  • Graphify stops blind grep loops where tokens get burned reading the wrong files.
  • Read-Website-Fast stops scraping garbage HTML (ads, banners, navigation) into context.
  • NotebookLM moves heavy retrieval off your API/quota bill entirely.
  • Awesome Design prevents redo loops on ugly UI; tokens you'd waste re-prompting.
  • ccr moves simple work to 10x-cheaper models (only matters when paying per token).

Even on a flat-fee MAX plan, the first four tools reduce context bloat and rate-limit hits. ccr is the only one tied specifically to API billing.
